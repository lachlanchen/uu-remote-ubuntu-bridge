# v0.2.0 - Backward-Compatible Union Release

Release date: 2026-07-18

This release combines the exact `v0.1.0` input fallback with the later
stability fixes, privacy-safe diagnostics, and optional tuning needed by more
difficult hosts. The old release remains available under its original,
immutable `v0.1.0` tag.

## Compatibility contract

The union is additive and conservative:

- ordinary mouse and physical-key input still calls UU's original `SendInput`
  path first and uses the proven normal-user broker only when that call fails
- upgrades from a `v0.1.0` environment preserve phone-text pacing at `0` ms
- new installations default phone-text pacing to 8 ms
- physical-key pacing defaults to `0` and is therefore opt-in
- network-interface filtering defaults to `all` and is therefore disabled
- saved port, resolution, display, keyring credential, Wine prefix, UU login,
  and unattended-startup choice are preserved
- unknown UU executable builds continue to fail closed

This means the known-good older computer can stay on `v0.1.0` indefinitely or
upgrade without silently inheriting this workstation's 12 ms physical-key
experiment.

## What the extended path adds

- bounded text and physical-key pacing without retrying or synthesizing keys
- foreground-focus confirmation before broker acknowledgement
- independent, content-free timing counters for text, keyboard, mouse, and
  other input calls
- a bridge-local libei keymap-descriptor fix and bounded GNOME RDP descriptor
  guard
- optional process-local network-interface selection that is fail-open and is
  not enabled by default
- installed-runtime digest verification and safer XRDP/client recovery notes

## Install or upgrade

```bash
cd ~/Projects/uu-remote-ubuntu-bridge
git status --short
git fetch --tags origin
git switch --detach v0.2.0
./install.sh --skip-packages --skip-account-login
./scripts/verify.sh --quick
```

Stop if `git status --short` reports local changes. Installation briefly
restarts the bridge but does not delete the Wine prefix or require another UU
login.

To opt into host-specific physical pacing later:

```bash
./install.sh --skip-packages --skip-account-login \
  --physical-key-delay-ms 8
```

Use `0` to restore the baseline behavior. Do not copy a tuned value to another
computer without a controlled test.

## Validation

- all 39 repository source, shell, documentation, and migration tests pass
- the ordinary input source still contains original-call-then-broker fallback
- installer tests require physical pacing `0` and network selection `all` as
  defaults
- migration tests require a pre-pacing environment file to resolve text delay
  to `0`
- the current workstation's saved `8` ms text and `12` ms physical values are
  preserved independently of repository defaults

## Rollback

The old working release is retained exactly:

```bash
git fetch --tags origin
git switch --detach v0.1.0
./install.sh --skip-packages --skip-account-login
./scripts/verify.sh --quick
```

Rollback replaces bridge components but preserves the UU account and Wine
prefix. `v0.1.0` does not understand the newer tuning fields; reinstalling
`v0.2.0` later reads them again from the persistent environment file.
