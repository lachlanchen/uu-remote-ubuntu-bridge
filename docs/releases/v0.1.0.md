# v0.1.0 - Supported Handoff Release

Release date: 2026-07-17

This is the first tagged release of UU Remote Ubuntu Bridge. It packages the
validated Ubuntu desktop relay, mobile keyboard correction, unattended startup,
and lifecycle hardening into one reproducible update point.

## Who should update

Update every installation based on commit `8a68200` or older. The update is
especially important when UU's phone keyboard types one repeated letter, such
as `d`, or turns number input into periods. UU's computer-keyboard panel can
still work in that state because it uses a different input path.

## Compatibility boundary

| Component | Supported release |
| --- | --- |
| Host | x86-64 Ubuntu 24.04 |
| Desktop | GNOME 46 on Wayland, Xorg, or XRDP |
| UU Remote | `4.33.0.8907` only |
| Wine | WineHQ stable 11 |
| Bridge | `v0.1.0` |

The installer rejects an unknown UU executable instead of applying old binary
edits to a new release.

## Highlights

### Phone keyboard normalization

The phone's native keyboard sends paired Windows events marked
`KEYEVENTF_UNICODE`. Wine presents those events to SDL FreeRDP as unsuitable
physical scancodes, which produced repeated letters and punctuation. The input
hook now routes every Unicode batch directly to the normal-user broker. The
broker maps each character through `VkKeyScanW`, emits an ordinary virtual-key
chord, and returns UU's original event count only when the complete translated
batch succeeds.

The hardware-keyboard path is unchanged. Diagnostic logs identify only whether
a batch was normalized; they never record the character values.

### Unattended and lifecycle operation

- the bridge starts from the user `default.target`
- opt-in GDM automatic login can unlock GNOME Keyring through a TPM2-encrypted
  systemd credential
- the live GNOME Shell display, Wayland socket, and D-Bus are discovered at
  runtime
- UU server replacement triggers compatibility DLL re-injection and account
  bootstrap
- any critical relay process exit rebuilds the complete relay through systemd
- Wine cleanup is bounded to the dedicated UU prefix

## Validation evidence

- 28 proprietary-binary-free repository tests pass locally and in GitHub
  Actions.
- A live 16-event `KEYEVENTF_UNICODE` batch returned its original count with
  `error=0` after normalization.
- The end-to-end local RDP relay rendered `aZ0.,!?` exactly in a GNOME text
  field.
- The complete runtime verifier kept one UU server PID stable for 270 seconds
  and passed every patch, process, account, input-broker, and RDP check.

## Update an existing direct clone

The repository is public. Existing clones can update without GitHub
authentication:

```bash
cd ~/Projects/uu-remote-ubuntu-bridge
git status --short
git fetch --tags origin
git checkout v0.1.0
./install.sh --skip-packages --skip-account-login
./scripts/verify.sh --quick
```

Stop before `git checkout` if `git status --short` prints local changes. The
installer briefly stops UU, rebuilds and installs the compatibility components,
preserves the Wine prefix and UU account state, then restarts and verifies the
service. Existing relay port, resolution, display selection, keyring secret,
and unattended drop-in remain in place.

## Update through The Art of Lazying

```bash
cd ~/Projects/the-art-of-lazying
git pull --ff-only origin main
git submodule sync --recursive
git submodule update --init --recursive
cd code/uu-remote-ubuntu-bridge
git describe --tags --exact-match
./install.sh --skip-packages --skip-account-login
./scripts/verify.sh --quick
```

The expected tag output is `v0.1.0`. The child repository is public, so the
submodule can be fetched without separate collaborator access.

## Acceptance test

Reconnect the phone after installation and type this exact string with the
normal phone keyboard:

```text
abcXYZ123,.!?
```

Then confirm the broker normalized the batch successfully:

```bash
log="$HOME/.local/share/wineprefixes/uu-remote/drive_c/users/$USER/Temp/uu-input-broker.log"
tail -80 "$log" | rg 'text=normalized .*result=[1-9][0-9]* error=0'
```

Run `./scripts/verify.sh` for the additional 270-second server-stability check.

## Known limits

- Phone text is validated for English characters representable by the active
  Wine keyboard layout. An unrepresentable character fails explicitly instead
  of becoming an unrelated key.
- Unattended startup remains opt-in because GDM automatic login exposes the
  desktop to anyone with physical access.
- The local RDP clipboard channel is enabled for normal copy and paste.
- No NetEase executable or prebuilt proprietary artifact is included.

## Rollback

If the release introduces a regression, restore the prior source state and
reinstall the previous components:

```bash
cd ~/Projects/uu-remote-ubuntu-bridge
git checkout 8a68200
./install.sh --skip-packages --skip-account-login
./scripts/verify.sh --quick
```

That rollback restores the previous bridge behavior, including its known
broken native phone-keyboard path. It does not delete UU account state. Use
`./uninstall.sh --dry-run` followed by `./uninstall.sh` only when removing the
bridge entirely.
